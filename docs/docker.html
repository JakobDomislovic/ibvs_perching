Run it in Docker

One container with everything prebuilt — ROS Noetic, Gazebo, ArduPilot SITL, mavros, the LARICS stack and this package. Three commands from git clone to a flying simulation.

Step 1

Prerequisites

You need a Linux host with Docker installed — nothing else. No ROS, no catkin workspace, no GitHub account or SSH key: the build clones only public repositories over HTTPS.

So that Gazebo, RViz and PlotJuggler can open windows on your screen, allow local containers to use your X server (once per login session):

xhost +local:docker

Optional, for NVIDIA GPU acceleration inside the container: install the nvidia-container-toolkit. Without a GPU everything still works — Gazebo just renders in software.

Step 2

Build the image

git clone https://github.com/JakobDomislovic/ibvs_perching.git
cd ibvs_perching
./docker/build.sh

The first build takes a while (~20–30 min) — it assembles the whole simulation world once, so that starting it later takes seconds. Under the hood, the Dockerfile:

  1. starts from the prebuilt LARICS uav_ros_stack binary image,
  2. clones uav_ros_simulation pinned to a known-good commit (ArduPilot SITL, ardupilot_gazebo, Gazebo worlds) and builds the simulation workspace,
  3. copies your local checkout of this package into the workspace and builds it — the image always matches the code you cloned.

No NVIDIA GPU? Build the plain-Ubuntu variant instead:

./docker/build.sh --nogpu
Step 3

Run the container and fly

./docker/run.sh          # add --enable-gpu if you built the GPU image and have the toolkit
./start.sh               # you land directly in startup/sim_ibvs — this starts everything

The container drops you straight into startup/sim_ibvs, so ./start.sh is the only command left — it brings up the whole tmux session (Gazebo, SITL, mavros, detector, controller). From here, continue with the simulation guide.

Detach from the container with Ctrl+P then Ctrl+Q. Running ./docker/run.sh again re-attaches to the existing container — your tmux session survives.

All flags

Command Effect
./docker/build.sh GPU-capable image (nvidia/opengl base) → ibvs_perching:focal
./docker/build.sh --nogpu plain Ubuntu 20.04 base → ibvs_perching:focal-nogpu
./docker/run.sh --enable-gpu adds --gpus all (needs nvidia-container-toolkit)
./docker/run.sh --nogpu runs the focal-nogpu image
./docker/run.sh --no-mount use the code baked into the image instead of the host checkout
./docker/run.sh --run-args "..." extra arguments forwarded to docker run
Step 4

Developing inside the container

By default run.sh mounts your checkout over the package inside the container (/root/sim_ws/src/ibvs_perching). Edit the Python nodes, launch files and configuration on the host with your favorite editor — the changes apply the next time you (re)launch the nodes, no image rebuild needed.

Only after changing CMakeLists.txt or package.xml, rebuild inside the container:

catkin build ibvs_perching

Flags apply only when the container is created

Mounts and GPU flags are set the first time run.sh creates the container; afterwards it always re-attaches to the existing one. To start fresh (e.g. to change mount options): docker rm ibvs_perching_focal, then ./docker/run.sh again.

Updating the pinned simulation

The LARICS simulation stack is pinned via the UAV_ROS_SIM_COMMIT build argument in the Dockerfile, so every student builds the exact same environment. To try a newer upstream commit:

./docker/build.sh --build-args "--build-arg UAV_ROS_SIM_COMMIT=<sha>"

If it works, make it permanent by updating the default value in the Dockerfile.

When things go wrong

Troubleshooting

Symptom Cause & fix
Gazebo window never appears / cannot connect to X server The container is not allowed to use your display. Run xhost +local:docker on the host (needed once per login session), then try again.
could not select device driver "nvidia" You passed --enable-gpu without the nvidia-container-toolkit installed. Install it, or simply run without the flag.
Changed a flag (mount, GPU) but nothing changed run.sh re-attached to the existing container — flags only apply on creation. docker rm ibvs_perching_focal and run again.
Edits on the host don't show up in the container You are running with --no-mount, or the container was created before the mount existed. Recreate it: docker rm ibvs_perching_focal → ./docker/run.sh.
Gazebo is very slow / camera image is black Software rendering. Build the GPU image (default build.sh), install the nvidia-container-toolkit and run with --enable-gpu.
