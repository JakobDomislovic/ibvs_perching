Flying with the keyboard joystick

Your keyboard becomes the RC transmitter of the simulated UAV.

Concept

What it is

keyboard_rc.py stands in for the real RC transmitter when flying in simulation. It does not fake anything inside the controller — it publishes real RC values on mavros/rc/override, so ArduPilot sees genuine stick input and even reports it back on mavros/rc/in. The same rc_to_joy bridge the real vehicle uses therefore works in simulation unchanged, and the whole manual-flight pipeline is identical to the field.

The keyboard drives the exact same RC pipeline a real transmitter would.

SYSID_MYGCS — handled for you

ArduPilot only accepts RC overrides from the ground station whose system id matches its SYSID_MYGCS parameter (factory default 255 = MAVProxy, while mavros is 1 — so by default every override is silently ignored). keyboard_rc.py sets SYSID_MYGCS = 1 itself at startup. Wait for the line “SYSID_MYGCS=1 — RC override enabled” before flying.

Reference

The keys

Start it in the ibvs tmux window (the command is pre-typed — press ↑ and Enter):

rosrun ibvs_perching keyboard_rc.py __ns:=$UAV_NAMESPACE

Keys act on whichever tmux pane has focus — keep the keyboard pane selected while flying. All four sticks are spring-loaded: a key deflects the stick, releasing it lets the stick snap back to center (hold the key for a continuous command — keyboard auto-repeat keeps it deflected). The bottom status line always shows the current mode, armed state, and stick values.

★ The IBVS button

I calls ibvs/start — the next tag detection engages the controller.
K calls ibvs/stop — stop servoing, hover.
This is the button the real joystick will have.

Flight modes

123G

1 STABILIZE · 2 ALT_HOLD · 3 LOITER — any of these = pilot takes over.
G GUIDED_NOGPS = hand control back.

Arming & utility

OPspace
HQ

O arm · P disarm (mid-air = crash!) · space center all sticks ·

Fly in ALT_HOLD, not STABILIZE

With a spring-loaded keyboard throttle, ALT_HOLD (2) is the mode to fly: centered throttle means “hold this altitude”, and W/S command climb/descent rates. In STABILIZE the throttle stick maps to raw thrust, which is very hard to manage with keys. While the vehicle is disarmed the throttle idles low automatically — ArduPilot refuses to arm with a raised throttle.

Tutorial

Your first flight, step by step

1
Start the sim (guide), open the ibvs window, select the keyboard pane, press ↑ + Enter. Wait for “SYSID_MYGCS=1 — RC override enabled”.
2
Press O to arm, then 2 for ALT_HOLD. The status line at the bottom should show ALT_HOLD  ARMED.
3
Hold W — the vehicle climbs. Release to hold altitude. Take it to 2–3 m.
4
Fly around with ←↑↓→ (tap for small corrections, hold for larger moves) and yaw with A/D. Watch the camera in rqt_image_view (ibvs/debug_image) and bring the ArUco marker into view — the tag sits at the world origin.
5
With the marker visible, press I. The controller announces the engagement, takes GUIDED_NOGPS, and ibvs/state jumps to ALIGN. Hands off — it centers the marker, then descends to 0.5 m above it and reports ALIGNED.
6
Practice the safety takeover: press 2 mid-alignment. You get the sticks back instantly, and the controller will not steal the mode back (the engagement is one-shot). Hand control back with G, or press I again to re-arm the engagement for the next detection.
7
Done flying? K stops the servoing (hover), P disarms on the ground, and Q quits the joystick, releasing all RC overrides.

Why the engagement is one-shot

After a pilot takeover the vision module usually keeps publishing detections. If the controller could re-engage on its own, pilot and software would fight over the flight mode — the most dangerous failure mode of a shared-control system. So the software mode switch is armed exactly once per button press: after a takeover, only you can hand control back (G or I). Details on the controller page.

Internals

Under the hood

  • Stick mapping. Keys set a deflection on one of four RC channels (roll, pitch, throttle, yaw; ±150 µs for attitude, ±250 µs for throttle around a 1500 µs center). A stick stays deflected while key auto-repeat keeps refreshing it and springs back ~0.6 s after the last press. Everything is parameterized (~step_tilt, ~step_throttle, ~hold_time, ~channel/*).
  • Modes and arming go through the regular mavros services (mavros/set_mode, mavros/cmd/arming) — exactly what a ground station would do.
  • The IBVS button (I / K) calls the controller's ibvs/start / ibvs/stop services. On the real vehicle the same services will be wired to a button on the RC transmitter.
  • Clean exit. Quitting publishes a full override release so ArduPilot returns to its own RC input, and the terminal is restored.
