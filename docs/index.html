About

What is this?

IBVS Perching is a small ROS (Noetic) package that lands a multirotor on a visual target using Image-Based Visual Servoing. A down-facing camera detects an ArUco marker, and a modal state-machine controller steers the vehicle until the marker is centered underneath it — then descends onto it.

The defining design choice: the controller talks to ArduPilot directly through mavros/setpoint_raw/attitude (body rates + a climb-rate command), completely bypassing the usual uav_ros_stack tracker/controller pipeline. That keeps the whole control path in one readable Python file and makes the package nearly self-contained.

The vehicle is flown manually with a joystick (a keyboard stand-in in simulation, a real RC in the field), and IBVS takes over only when the pilot presses the IBVS button — the same interaction planned for the real experiments.

System

How the pieces connect

Everything runs inside the /red UAV namespace. The pilot flies through mavros/rc/override; the controller flies through mavros/setpoint_raw/attitude. Pressing the IBVS button hands control from one path to the other.

The two paths never fight each other: RC sticks matter in the manual modes (STABILIZE, ALT_HOLD, LOITER), attitude setpoints matter only in GUIDED_NOGPS. Switching the flight mode is the handover — and the safety pilot can always take the mode back.

Workflow

A mission at a glance

The intended flow — identical in simulation and (soon) in the field:

1
Fly manually. The pilot arms and flies the vehicle with the joystick (keyboard in simulation, RC transmitter in the field). The controller is already running, but its setpoints are ignored in manual modes.
2
Find the target. The down-facing camera picks up the ArUco marker and the vision module starts publishing ibvs/target_point. Nothing takes over yet — seeing the tag alone never grabs control.
3
Press the IBVS button (I in simulation). The next tag detection engages: the controller switches the flight mode to GUIDED_NOGPS by itself and starts servoing — no position_hold handshake, no extra services.
4
Watch it align and descend. The controller centers the marker, then descends toward it while a safety gate keeps the descent centered. The pilot can take back control at any moment with the mode switch — the controller never steals the mode back on its own.
